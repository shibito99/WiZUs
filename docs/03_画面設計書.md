## 2. 画面別詳細

### S-1: ユーザー登録画面 `/register`

**目的:** 新規ユーザーがアカウントを作成する

**入力項目:**
| フィールド | 型 | バリデーション |
|-----------|-----|-------------|
| ユーザー名 | text | 必須・2〜50文字 |
| メールアドレス | email | 必須・メール形式 |
| パスワード | password | 必須・8文字以上 |

**操作:**
- 「登録する」ボタン → `/api/auth/register` POST → 成功時 JWT 保存 → `/` へリダイレクト
- 「ログインはこちら」リンク → `/login` へ遷移

**エラー表示:** フィールド単位でインラインエラーを表示（例: メール重複・パスワード短すぎ）

---

### S-2: ログイン画面 `/login`

**目的:** 既存ユーザーが認証してアプリを利用する

**入力項目:**
| フィールド | 型 | バリデーション |
|-----------|-----|-------------|
| メールアドレス | email | 必須 |
| パスワード | password | 必須 |

**操作:**
- 「ログイン」ボタン → `/api/auth/login` POST → 成功時 JWT 保存 → `/` へリダイレクト
- 「新規登録はこちら」リンク → `/register` へ遷移

**エラー表示:** 認証失敗時はフォーム上部に「メールアドレスまたはパスワードが正しくありません」を表示

---

### S-3: タイムライン画面 `/`

**目的:** フォロー中ユーザーの最新投稿を確認し、リアクションする

**表示コンテンツ:**
- ナビゲーションバー（検索・投稿ボタン・プロフィールリンク）
- PostCard の一覧（新着順、20件/ページ）
- ページネーション

**PostCard コンポーネント:**
| 要素 | 内容 |
|------|------|
| アバター | ユーザーのアバター画像（S3 URL） |
| ユーザー名 | クリックでプロフィール画面へ |
| 投稿日時 | 相対表示（例: 3分前） |
| 本文 | テキスト（最大280文字） |
| 画像 | 添付画像（任意） |
| いいね！ボタン | ♥ + 件数。トグルで色変化 |
| コメントボタン | 💬 + 件数。クリックでコメントモーダルを開く |
| 削除ボタン | 自分の投稿のみ表示 |

**コメントモーダル:**
- コメント一覧（投稿者・本文・日時・返信一覧）
- コメント入力フォーム
- 各コメントに「返信する」ボタン

---

### S-4: 投稿作成画面 `/posts/new`

**目的:** テキストと画像を投稿する

**入力項目:**
| フィールド | 型 | バリデーション |
|-----------|-----|-------------|
| 本文 | textarea | 必須・最大280文字（文字数カウンター表示） |
| 画像 | file | 任意・JPEG/PNG・10MB 以下 |

**操作:**
- 画像選択 → プレビュー表示
- 「投稿する」ボタン → `/api/posts` POST (multipart/form-data) → 成功時 `/` へリダイレクト
- 「キャンセル」リンク → `/` へ戻る

---

### S-5: プロフィール画面 `/users/[id]`

**目的:** 自分・他人のプロフィールと投稿一覧を確認する

**表示コンテンツ:**
| 要素 | 内容 |
|------|------|
| アバター | 画像（S3 URL） |
| ユーザー名 | — |
| 自己紹介 | bio テキスト |
| フォロー中 | 件数（クリックで `/users/[id]/following`） |
| フォロワー | 件数（クリックで `/users/[id]/followers`） |
| フォローボタン | 他ユーザー閲覧時に表示。フォロー中 / フォローする を切り替え |
| プロフィール編集ボタン | 自分のプロフィール時のみ表示 |
| 投稿一覧 | そのユーザーの PostCard 一覧（20件/ページ） |

---

### S-6: フォロー一覧画面 `/users/[id]/following`

**目的:** 指定ユーザーのフォロー中ユーザー一覧を確認する

**表示コンテンツ:**
- UserCard の一覧（アバター・ユーザー名・自己紹介・フォロー/アンフォローボタン）

---

### S-7: フォロワー一覧画面 `/users/[id]/followers`

**目的:** 指定ユーザーのフォロワー一覧を確認する

**表示コンテンツ:**
- UserCard の一覧（アバター・ユーザー名・自己紹介・フォロー/アンフォローボタン）

---

### S-8: ユーザー検索画面 `/search`

**目的:** ユーザー名でユーザーを検索し、知り合い候補からフォローする

**表示コンテンツ:**

**[検索セクション]**
- 検索ボックス（入力と同時に `/api/users/search?q=` にリクエスト）
- 検索結果 UserCard 一覧

**[知り合いかも？セクション]**
- 画面初期表示時に `/api/users/suggestions` を取得して表示
- UserCard 一覧（最大10件）
- 各カードの「フォローする」ボタンでワンクリックフォロー
- フォロー後はボタンが「フォロー中」に変わり、リストから除去

## 3. 共通コンポーネント

| コンポーネント | 用途 |
|-------------|------|
| `Navbar` | 全画面共通。ロゴ・検索リンク・投稿ボタン・プロフィールアイコン |
| `PostCard` | タイムライン・プロフィール画面で使用 |
| `CommentModal` | PostCard から呼び出すコメント・返信モーダル |
| `UserCard` | フォロー一覧・フォロワー一覧・検索画面で使用 |
| `LikeButton` | PostCard 内いいねトグルボタン |
| `FollowButton` | UserCard・プロフィール画面で使用 |
| `Avatar` | アバター画像表示（S3 URL またはデフォルトアイコン） |
