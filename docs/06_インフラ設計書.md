## 2. AWS リソース詳細

### EC2

| 項目 | 値 |
|------|-----|
| インスタンスタイプ | t3.small |
| OS | Amazon Linux 2023 |
| 配置 | Public Subnet (AZ-A) |
| 役割 | Spring Boot アプリサーバー + Nginx |
| IAM Role | SNSAppEC2Role（S3 読み書き権限） |
| Elastic IP | 付与（ALB の背後のため省略可） |

**ポート構成:**
| ポート | 用途 |
|--------|------|
| 8080 | Spring Boot（ALB からのみ受信） |
| 80 | Nginx（ALB からのみ受信） |

---

### RDS

| 項目 | 値 |
|------|-----|
| エンジン | MySQL 8.0 |
| インスタンスクラス | db.t3.micro |
| ストレージ | gp3 / 20GB |
| Multi-AZ | 有効（Primary: AZ-A / Standby: AZ-B） |
| バックアップ保持期間 | 7日間 |
| 配置 | Private Subnet |
| 接続ポート | 3306 |
| 接続許可 | EC2 のセキュリティグループからのみ |

---

### S3

| 項目 | 値 |
|------|-----|
| バケット名 | sns-app-images |
| リージョン | ap-northeast-1 |
| ストレージクラス | Standard |
| パブリックアクセス | ブロック（全て） |
| アクセス方法 | IAM Role 経由（EC2 → SDK） |

**フォルダ構成:**
```
sns-app-images/
  ├── posts/      投稿画像（posts/{uuid}-{filename}）
  └── avatars/    アバター画像（avatars/{uuid}-{filename}）
```

---

### ALB (Application Load Balancer)

| 項目 | 値 |
|------|-----|
| タイプ | Application |
| スキーム | インターネット向け |
| リスナー | HTTP:80 → HTTPS:443 リダイレクト |
| リスナー | HTTPS:443 → EC2:8080 転送 |
| SSL 証明書 | ACM（AWS Certificate Manager）で発行 |
| ターゲットグループ | EC2 (Port 8080)、ヘルスチェック: `GET /actuator/health` |

---

### Route 53

| 項目 | 値 |
|------|-----|
| ホストゾーン | example.com |
| レコード | A レコード（エイリアス） → ALB |

---

### VPC / ネットワーク

| リソース | CIDR / 値 |
|---------|-----------|
| VPC | 10.0.0.0/16 |
| Public Subnet (AZ-A) | 10.0.1.0/24 |
| Private Subnet (AZ-A) | 10.0.2.0/24 |
| Private Subnet (AZ-B) | 10.0.3.0/24 |
| Internet Gateway | VPC にアタッチ |
| NAT Gateway | Public Subnet に配置（EC2 からの Outbound 用） |

---

### セキュリティグループ

**ALB-SG:**
| 方向 | プロトコル | ポート | ソース |
|------|-----------|--------|--------|
| Inbound | TCP | 80 | 0.0.0.0/0 |
| Inbound | TCP | 443 | 0.0.0.0/0 |

**EC2-SG:**
| 方向 | プロトコル | ポート | ソース |
|------|-----------|--------|--------|
| Inbound | TCP | 8080 | ALB-SG |
| Outbound | TCP | 3306 | RDS-SG |
| Outbound | TCP | 443 | 0.0.0.0/0 (S3 アクセス) |

**RDS-SG:**
| 方向 | プロトコル | ポート | ソース |
|------|-----------|--------|--------|
| Inbound | TCP | 3306 | EC2-SG |

## 3. デプロイ構成

### バックエンド（Spring Boot）

```
1. mvn package -DskipTests
2. scp target/backend-0.0.1-SNAPSHOT.jar ec2-user@<EC2-IP>:~/app/
3. sudo systemctl restart sns-app
```

**systemd サービスファイル** `/etc/systemd/system/sns-app.service`:
```ini
[Unit]
Description=TextSNS Spring Boot App

[Service]
User=ec2-user
ExecStart=/usr/bin/java -jar /home/ec2-user/app/backend-0.0.1-SNAPSHOT.jar
EnvironmentFile=/home/ec2-user/app/.env
Restart=always

[Install]
WantedBy=multi-user.target
```

### フロントエンド（Next.js）

```
1. npm run build
2. Nginx で .next/static を配信
   または out/ を静的エクスポートして Nginx から配信
```

## 4. 環境変数

### バックエンド（`/home/ec2-user/app/.env`）

| 変数名 | 説明 |
|--------|------|
| `DB_USERNAME` | RDS MySQL ユーザー名 |
| `DB_PASSWORD` | RDS MySQL パスワード |
| `DB_URL` | `jdbc:mysql://<RDS-endpoint>:3306/sns_db` |
| `JWT_SECRET` | JWT 署名秘密鍵（256bit 以上） |
| `S3_BUCKET_NAME` | `sns-app-images` |
| `AWS_REGION` | `ap-northeast-1` |
