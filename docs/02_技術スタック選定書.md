## 2. アーキテクチャ選定の背景

### フロントエンド: Next.js + TypeScript + Tailwind CSS

初級課題で同構成を採用・習得済みのため、学習コストゼロで開発に集中できる。
Spring Boot（REST API）と Next.js（フロントエンド）を分離する構成により、責務が明確になる。

### バックエンド: Spring Boot

RaiseTech 中級課題のテーマとして指定。Java + Spring のエコシステムで認証・DB・ファイルアップロードを統合管理できる。

### DB: MySQL（RDS）

フォロー・いいね・コメントといった多対多・親子関係のデータを持つため、リレーショナル DB が適合する。
DynamoDB では JOIN が複雑になる一方、MySQL + JPA では自然に表現できる。

### ストレージ: S3

初級課題で Presigned URL による S3 画像アップロードを経験済み。
今回はサーバサイド（Spring Boot）経由でのアップロードに変更し、アクセス制御を強化する。
