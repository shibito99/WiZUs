# 要件定義書

**プロジェクト:** TextSNS
**作成日:** 2026-07-11
**ステータス:** 確定

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| サービス名 | TextSNS |
| 目的 | テキスト・画像を投稿し、ユーザー同士がつながれる SNS サービス |
| 参考モデル | Twitter/X に近い UX |
| 対象ユーザー | 登録制。未ログインユーザーは一部コンテンツの閲覧のみ可 |

---

## 2. 機能要件

### 2-1. ユーザー管理

| # | 機能 | 詳細 |
|---|------|------|
| U-1 | ユーザー登録 | ユーザー名・メールアドレス・パスワードで登録 |
| U-2 | ログイン | メール＋パスワード認証 → JWT 発行 |
| U-3 | ログアウト | クライアント側でトークン破棄 |
| U-4 | プロフィール編集 | アバター画像（S3）・自己紹介文の更新 |
| U-5 | ユーザー検索 | ユーザー名での部分一致検索 |
| U-6 | 知り合いかも？ | フォロー中ユーザーが共通してフォローしている人を最大10件提案。フォロー0人の場合はフォロワー数上位ユーザーを表示。ワンクリックでフォロー可能 |

### 2-2. 投稿

| # | 機能 | 詳細 |
|---|------|------|
| P-1 | 投稿作成 | テキスト（最大280文字）＋画像1枚（任意） |
| P-2 | 投稿削除 | 自分の投稿のみ削除可 |
| P-3 | タイムライン | フォロー中ユーザーの投稿を新着順・ページネーション（20件/ページ）。フォロー0人の場合は自分の投稿のみ表示 |
| P-4 | いいね！ | 投稿へのいいね／取り消し（トグル）、件数表示 |
| P-5 | コメント | 投稿へのコメント追加・削除（自分のみ）、件数表示 |
| P-6 | 返信 | コメントへの返信（1階層まで） |

### 2-3. フォロー

| # | 機能 | 詳細 |
|---|------|------|
| F-1 | フォロー / アンフォロー | 他ユーザーをフォロー・解除 |
| F-2 | フォロー一覧 | 自分・他人のフォロー中ユーザー一覧 |
| F-3 | フォロワー一覧 | 自分・他人のフォロワー一覧 |
| F-4 | フォロー数表示 | プロフィール画面にフォロー数・フォロワー数を表示 |

---

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| 認証 | JWT（有効期限 24 時間） |
| 画像サイズ上限 | 10 MB |
| 対応ブラウザ | Chrome 最新版 |
| API 形式 | REST / JSON |
| CORS | フロントエンドオリジン（開発: localhost:3000、本番ドメイン）のみ許可 |

---

## 4. ユースケース一覧

| # | ユースケース | アクター | 概要 |
|---|------------|---------|------|
| UC-1 | 新規登録・ログイン | 未登録ユーザー | 登録 → JWT 取得 → タイムラインへ遷移 |
| UC-2 | 投稿する | ログイン済みユーザー | テキスト＋任意画像を投稿・削除 |
| UC-3 | タイムラインを見る・リアクションする | ログイン済みユーザー | いいね・コメント・返信・件数確認 |
| UC-4 | 他のユーザーを探してフォローする | ログイン済みユーザー | 検索・知り合いかも？からワンクリックフォロー |
| UC-5 | プロフィールを確認・編集する | ログイン済みユーザー | アバター・自己紹介更新・他ユーザー閲覧 |
| UC-6 | フォロー関係を確認する | ログイン済みユーザー | フォロー/フォロワー一覧・フォロー操作 |

詳細なユースケース記述は [README.md](../README.md) を参照。

---

## 5. 制約・前提条件

- 画像は1投稿につき1枚まで
- コメントの返信は1階層のみ（返信への返信は不可）
- タイムラインはフォロー中ユーザーの投稿のみ表示（全体公開タイムラインなし）
- 認証が必要な操作は JWT がない場合 401 を返す
